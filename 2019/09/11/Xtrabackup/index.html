Percona XtraBackup安装实践

发表于 2019-09-11

简介

Percona XtraBackup 是开源免费的 MySQL 数据库热备份软件，它能对 InnoDB 和 XtraDB 存储引擎的数据库非阻塞地备份（对于 MyISAM 的备份 同样需要加表锁）。XtraBackup 支持所有的 Percona Server、MySQL、MariaDB 和 Drizzle。

物理备份

XtraBackup优势 ：

1、无需停止数据库进行 InnoDB 热备，备份过程中不会锁表，不影响现有业务。

2、增量备份 MySQL。

3、流压缩到传输到其它服务器。

4、能比较容易地创建主从同步。

5、备份MySQL时不会增大服务器负载。

原理

Xtrabackup 包括两个工具: xtrabackup、innobackupex ，其主要区别如下:

  • xtrabackup: 只能备份innodb和xtradb两种引擎的表，不能备份myisam引擎的表
  • innobackupex: 是一个封装了xtrabackup的Perl脚本，支持同时备份innodb和myisam，但在对myisam备份时需要加一个全局的读锁。还有就是myisam不支持增量备份。

文件功能

Percona Xtrabackup 备份生成的几个文件功能说明

Percona Xtrabackup 备份过程中，生成的几个文件功能说明

  • backup-my.cnf : 启动备份集所需要的最小参数
  • xtrabackup_binlog_info : 备份完成的时间点的 binlog 位点
  • xtrabackup_checkpoints : 备份类型是全量备份还是增量备份，当前备份集的状态，是否已经prepare过。如果是全量备份，backup_type = full-backuped，如果是增量是backup_type = incremental。全量备份备份集的状态在 apply log 后由 full-backuped 变成了 full-prepared
  • xtrabackup_info 记录备份的时候的一些参数，脚本版本，数据库版本，备份时间，binlog，是否压缩，备份增量还是全量等信息
  • xtrabackup_binlog_pos_innodb : apply log完成后生成,记录的 innodb 和 xtradb 的 binlog 位点，所以如果存在多种存储引擎，需要利用 xtrabackup_binlog_info 来确定备份的完成那个时刻的 binlog 位点.
  • xtrabackup_logfile : 记录从备份开始到备份结束所有的 redo 日志，xtrabackup 在 apply log 的时候需要读取该文件，进行 redo 前滚。
  • xtrabackup_slave_info : 记录的是主库的 binlog 信息，该信息可以用来通过从库创建一个从库，然后指向主库！非常棒！该信息需要在备份的时候加上 –slave-info .

其他问题

问题一

问题现象

使用yum 命令安装依赖包percona-xtrabackup-24-2.4.12-1.el7.x86_64.rpm时报错

报错提示mariadb-libs 与之前安装的 MySQL 发生冲突，查看 MySQL 安装包目录

检查后发现报错是由MySQL组件未安装完全导致的，之前安装MySQL时是通过 RPM 包的方式，为了省事只安装了部分组件便直接使用，未安装 MySQL 的shared 和embedded 组件。

解决方法

为了方便，将未安装的 MySQL 组件安装上

将报错的yum命令重新执行，查看xtrabackup版本

安装成功

问题二

问题现象

执行rpm 命令安装xtrabackup时报错冲突。

解决办法

服务器上安装过Percona XtraBackup2.2, 但从mysql5.7开始，必须使用xtrabackup2.4以上版本才能完成备份,老的版本执行会报错，已不适用。将老版本卸载后重新安装.

问题三

问题现象

解决方法：

查看操作系统版本，CentOS 6 下载 el6 的安装包即可。

Tips：不要轻易在真实生产环境中直接安装，因为升级的过程中比较容易出现系统出错。
