案例2 MySQL CPU利用率过高

发表于 2019-09-11

问题现象

CPU利用率连续15分钟超过了（90%）达到96.75885416666917%，请求协助排查原因。

原因分析

1)连接上服务器使用top命令动态查看cpu的状态发现MySQL CPU使用率长时间处于异常过高的状态。

2）使用show processlist 查看数据库线程，未发现有锁占用和大量线程阻塞的情况。

3）查看数据库日志未发现明显异常

4）重新查看数据库线程，发现有许多查询都处于creating sort index的状态，通过客户端抽取其SQL语句，都是如下类似的查询结构：

除了material编号发生变化，其余部分相同。

分析上述SQL查询语句的执行逻辑：
