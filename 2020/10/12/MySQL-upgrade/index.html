MySQL5.5 -> 5.7升级

发表于 2020-10-12

操作步骤

关闭 MySQL 服务

下载新版本 MySQL 安装包，并解压至 /user/local/ 目录下

取消旧版本 mysql 的软连接，并为新版本 mysql 软连接

启动新版本的 MySQL 服务，可能会出错，因为旧版本的某些参数被新版本抛弃，出错后查看日志信息把对应的参数设置从/etc/my.cnf中注销，然后重新启动 MySQL 服务即可

5、升级系统表数据字典信息，该过程中会出现许多 error ，提示“ALTER TABLE ** FROCE”，这些错误不用担心mysql在后面会自动修复，但是修复时间可能会有点长，当出现“Upgrade process completed successfully.”说明升级成功

6、升级完以后重启一下 MySQL

问题

在升级 mysql5.5到 mysql5.7 的时候，替换 mysql 的 base 目录后，用mysqld_safe 命令无法启动，错误日志无记录：

尝试使用 mysqld 直接启动

控制台输出：

升级GLIBCXX

重新使用mysqld_safe 启动，启动成功。

总结

  • 根因：当前的 GCC 版本中，没有 GLIBCXX_3.4.15 ，须要安装更高版本。

    mysql_safe 启动数据库会调用 mysqld ,然后 MySQL5.7 的 mysqld 命令需要 GLIBCXX_3.4.15

  • 遇到需要移动 data 目录的情况，可通过创建软链接的方式直接指向新的目录，不要直接移动。

  • ibdata1 和 ibdata2 文件存放了表结构，不可直接删除
